# Code of Conduct

Peko Agent is a small technical project, not a social one. The standards here
are light but mandatory.

## In scope

This Code of Conduct applies to:

- GitHub issues, discussions, and pull request reviews on this repository.
- Any chat channel, mailing list, or event officially tied to the project.
- Direct communications with maintainers acting in their project capacity.

## Expected behaviour

- **Be kind to people, harsh on code.** Critique the patch, not the author.
- **Assume competence.** Contributors have their own context — ask before
  correcting.
- **Stay on topic.** Bug reports debate code, not politics, not personalities.
- **Disclose conflicts of interest.** If you're filing a PR that benefits
  your employer or product, say so in the PR description.

## Unacceptable behaviour

- Harassment, personal attacks, slurs, or demeaning jokes — including ones
  directed at other open-source communities.
- Publishing other people's private information (doxxing) without explicit
  consent.
- Sustained disruption of discussion — spamming issues, derailing PR threads,
  pinging maintainers past a reasonable cadence for a reply.
- Sexualised attention or advances in any project space.
- Anything that would get a normal person kicked out of a normal engineering
  meeting.

## Security + abuse of the project itself

Peko Agent runs with root on people's phones, sends SMS, places calls, and
records audio. A pattern of "contributions" that try to quietly weaken these
guardrails — removing rate limits, stripping consent beeps, silently
exfiltrating data, adding telemetry without opt-in — is treated as a Code of
Conduct violation, not just a bad PR. Those PRs will be closed and the author
banned from the project.

Vulnerabilities you discover while using the project go to
[`SECURITY.md`](SECURITY.md), not to public issues.

## Reporting

Incidents that happen in issues / PRs are best handled in the open: mention
what you think is wrong, tag a maintainer. If that's not appropriate — for
example, it involves you personally and would escalate if public — email
**tanuphat.chai@gmail.com** with a brief summary. Reports are read by the
maintainer and kept confidential.

Please don't report something just because you disagree with a technical
decision. "I wanted my feature, they said no" is not a Code of Conduct issue.

## Enforcement

The maintainers may, depending on severity:

1. Ask you to edit or delete the offending comment (minor first-time friction).
2. Lock the thread and continue in private.
3. Temporarily block the account from the repository.
4. Permanently ban the account from the repository and related channels.

Decisions are final. There's no appeals panel on a two-person project — if
you think a decision was wrong, you're welcome to fork.
